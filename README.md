# 🪟 Panedows-XP

**Panedows-XP** is a chaotic, browser-based parody of Windows XP created for a hackathon with the theme **“Chaos Creations.”**

> 🥈 **2nd Place Winner**  
> ⏱ Built in ~10 hours (including planning)

> Team Credits:
- Integrations / Scripting: https://github.com/SIndustryPortfolio
- Design / UI & UX: https://github.com/schrls-designs

The project simulates a desktop operating system inside the browser while intentionally introducing frustration, delays, and absurd behaviour — turning the user experience into a slightly painful one.

**Pane** as in *window*.  
**Pain** as in *the experience.*

## Concept

Panedows-XP recreates a Windows-XP-style desktop environment but exaggerates the worst parts of old operating systems:

- Slow loading
- Random errors
- Overdramatic system feedback

All while running entirely client-side.

---

## Design Features

- ✏️ Hand-drawn icons
- 🧩 Component-based UI layout
- 📱 Responsive / reactive windows when resized
- 🖥 Fullscreen support for immersion

---

## Technical Highlights

- Organised project file structure
- React-style component architecture
- OOP services & modular system design
- Game-dev inspired runtime loops
- Tween / LERP animations using JS + CSS

---

## Features

- Desktop-style **apps**
- Native browser calls (e.g. Printer app → `window.print()`)
- Cursor loading animation when opening apps
- Audio cues & visual animations
- Error pop-ups
- **Blue Screen of Death**
- Boot screen that intentionally slows near completion
- Indefinite boot loop

---

## Hackathon Theme

**Chaos Creations**

Panedows-XP embraces chaos in the **user experience**, while keeping the **code architecture structured and modular**.

> The UI is chaos.  
> The code is not.
